Docker

Docker — это открытая платформа для автоматизации развертывания, доставки и управления приложениями в контейнерах. Она позволяет упаковать приложение со всеми его зависимостями в единый контейнер, который может быть развернут на любой поддерживаемой операционной системе.

  • Docker image (образ) — это файл, который содержит всё необходимое для запуска приложения или сервиса, включая код программы, зависимости, настройки и окружение. Образы можно использовать для создания и запуска контейнеров.

    Docker container (контейнер) — это экземпляр, или «экземпляр выполнения», Docker image. Он представляет собой окружение, в котором запускается приложение из Docker image.

    Образно говоря, Docker image (образ) — это рецепт, по которому готовится блюдо. Docker container (контейнер) — это само приготовленное блюдо, которое может быть размещено на столе и съедено.

  • Docker Hub — это реестр готовых докер-образов. Можно загружать свои для совместной работы, можно скачивать образы, сделанные другими. Аналог GitHub, но для Docker-образов.

Основные команды Docker:

  • docker pull <image_name> — загрузка докер-образа (Docker image)
  • docker images — просмотр загруженных образов
  • docker run -dp <port> <image_name> — запуск контейнера
  • docker ps — просмотр списка запущенных контейнеров (Docker container)
  • docker stop <container id> — остановка контейнера
  • docker restart <container id> — перезапуск контейнера
  • docker rm <container id> — удаление контейнера
  • docker image rm —force <image id> — удаление образа
